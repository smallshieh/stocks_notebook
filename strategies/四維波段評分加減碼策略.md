# 四維波段評分加減碼策略（Wave Score 4D）

> 建立日期：2026-03-12
> 工具腳本：`scripts/wave_position.py`
> 適用：所有持倉與候補標的的波段位置判斷、加碼 / 減持決策支援

---

## 核心理念

傳統的「看 K 線決定買賣」方式過於主觀。本策略從四個**量化維度**各自打分，加總後形成客觀的波段位置評分（Wave Score），再對應操作建議，**消除情緒性決策**。

---

## 評分系統

每個維度分數：-2 ～ +2（或 +3）

| 維度 | 工具 | 說明 | 分數範圍 |
|------|------|------|---------|
| **1. 均線結構** | MA 排列 | 現價 > 5MA > 10MA > 20MA > 60MA 各 +1，總分 0~4，基準 -2 | -2 ～ +2 |
| **2. GBM σ 位置** | GBM 漂移模型 | 現價相對 20日 GBM 期望值的 σ 距離 | -2 ～ +2 |
| **3. 分位數區間** | 歷史回檔分位 | 現價在超漲收割 / 合理 / 常規買回 / 深回檔哪一區 | -2 ～ +3 |
| **4. 物理引擎** | 動量 / 雷諾數 / 預警 | 動量方向、流場狀態、反重力 / 能量耗散預警 | -2 ～ +2 |

### 分維度評分細則

**均線結構（MA）：**
| 條件符合數 | 原始分 | 評分 |
|----------|--------|------|
| 4/4（完整多頭排列）| 4 | +2 |
| 3/4 | 3 | +1 |
| 2/4 | 2 | 0 |
| 1/4 | 1 | -1 |
| 0/4（完整空頭排列）| 0 | -2 |

**GBM σ 位置（20日）：**
| 現價區間 | 評分 |
|---------|------|
| < 期望值 - 0.5σ（低估）| +2 |
| 期望值 ± 0.5σ（合理）| 0 |
| 期望值 + 0.5σ ～ +1σ（偏貴）| -1 |
| > 期望值 + 1σ（極端貴）| -2 |

**分位數區間：**
| 現價落點 | 評分 |
|---------|------|
| 深回檔加碼區 | +3 |
| 常規買回區 | +2 |
| 合理持有區 | 0 |
| 超漲收割區 | -2 |
| 低於暫停線 | -3 |

**物理引擎（4 項加總，基準 -2）：**
| 條件 | 說明 |
|------|------|
| 動量正向 ↑ | +1 |
| 雷諾數 < 2000（非極端湍流）| +1 |
| 無反重力預警 | +1 |
| 無能量耗散預警 | +1 |
| 以上每符合一項 +1，總和 -2（基準校正），結果 -2 ～ +2 | |

## 操作建議閾值

| Wave Score | 建議 | 操作方向 |
|-----------|------|---------|
| ≥ +5 | 🟢 強力加碼 | 全力建倉 / 加碼至上限 |
| +3 ～ +4 | 🟢 加碼 | 常規加碼 |
| +1 ～ +2 | 🟡 輕倉加碼 / 觀察 | 試倉或持有 |
| -1 ～ 0 | 🟡 持有不動 | 不動作 |
| -2 ～ -3 | 🔴 部分減持 | 波段倉減碼（底倉不動）|
| ≤ -4 | 🔴 強力減持 | 考慮大幅減倉 |

> ⚠️ **注意**：Wave Score 是輔助工具，**不是交易訊號**。
> 對於長線趨勢股（台積電、鴻海），即使 Score 為負，底倉依然不動；
> Score 指導的是**波段調節倉**的加減碼時機。

## 情境與攻防動作對照

> 用途：把 Wave Score 從「技術分數」轉成「可執行動作」。  
> 判斷順序固定為：**風險邊界 → 持倉定位 → 價格位置 → Wave Score → 攻防動作**。

### 0. 風險邊界優先

若下列任一條件成立，直接執行原本風控，不再用 Wave Score 覆蓋：

| 觸發 | 動作 |
|------|------|
| 觸及硬停損 / 時間停損 | 依該檔交易檔全出或減持 |
| P0 / P1 出清計畫已啟動 | 優先出清，不加碼 |
| Cash 低於保底線 | 禁止新建倉與非必要加碼 |
| 單檔超過預算上限 | 禁止加碼，反彈優先降倉 |
| 停損回收資金剛入帳 | 不立即再投入同檔 |

### 1. 持倉定位轉譯

同一個 Wave Score，因持倉定位不同，動作不同：

| 持倉定位 | Wave 正分 | Wave 中性 | Wave 負分 |
|---------|-----------|-----------|-----------|
| 🔒 Core 底倉 | 可小額加碼，但需基本面未壞 | 持有，不因短線震盪賣底倉 | 底倉不動；只檢查基本面是否轉壞 |
| 🎯 Tactical 波段倉 | 依計畫加碼或續抱 | 停止加碼，觀察收復月線 | 減持波段倉，Wave ≤ -2 優先防守 |
| ⚙️ 零股滾動操作倉 | 不追高；等賣出區執行賣高 | 按 Phase 計畫，不主觀加碼 | 到買回區才按計畫買回，否則不動 |
| 🚪 減損出清倉 | 反彈找賣點，不加碼 | 等反彈或依硬死線處理 | 加速出清，不攤平 |
| 🆕 新倉 / 候補 | 可試單，但須有出場線 | 等月線 / 量能確認 | 暫停，不進場 |

### 2. 跌破月線後的攻防表

| 情境 | 判斷 | 動作 |
|------|------|------|
| 跌破月線 < 2%，Wave ≥ 0，Core / 底倉 | 月線附近整理 | 觀察，不賣底倉 |
| 跌破月線 < 2%，Wave ≥ +2，Tactical | 可能是假跌破，但仍未確認 | 不追買，等收復月線再加碼 |
| 跌破月線 3%～5%，Wave 0～+1 | 趨勢轉弱但未崩壞 | 停止加碼，保留觀察 |
| 跌破月線 3%～5%，Wave ≤ -2 | 波段轉弱確認 | Tactical 減持 1/3～1/2 |
| 跌破月線 > 5%，Wave ≤ -3 | 波段破壞 | Tactical 全出或依停損線處理 |
| 跌破月線但進入買回區，Core 且基本面未壞 | 上升趨勢回檔 | 可小額加碼 |
| 跌破月線但進入買回區，減損出清倉 | 下跌趨勢反彈前段 | 不加碼，等反彈賣 |
| 跌破月線且觸及硬停損 / 時間停損 | 風險邊界觸發 | 直接執行停損 |

### 3. 賣出區的攻防表

| 情境 | 判斷 | 動作 |
|------|------|------|
| 進入賣出區，Wave ≥ +2 | 趨勢延伸，尚未背離 | 續抱；等爆量收黑、Wave 降溫或停利線 |
| 進入賣出區，Wave 0～+1 | 動能不足 | 波段倉減 1/3，Core 不動 |
| 進入賣出區，Wave ≤ 0 且腳本標記動能背離 | 高位轉弱 | Tactical 減半或依計畫停利 |
| 賣出區 + 已超倉 | 風險報酬不佳 | 優先降到預算內 |
| 賣出區 + 減損出清倉反彈 | 反彈逃命 | 按出清計畫賣出 |

### 4. 買回區的攻防表

| 情境 | 判斷 | 動作 |
|------|------|------|
| 買回區，Wave ≥ +2，Core / 長期趨勢股 | 回檔後動能仍在 | 可小額加碼 |
| 買回區，Wave 0～+1 | 價格合理但動能普通 | 觀察或試單，不重押 |
| 買回區，Wave ≤ -2 | 價格便宜但趨勢破壞 | 不加碼，等修復 |
| 買回區，零股滾動操作倉 | 策略還原點 | 按 Phase 買回，不視為新加碼 |
| 買回區，減損出清倉 / P1 | 下跌途中的便宜 | 不加碼 |

### 5. 行動強度

| 訊號組合 | 行動強度 |
|----------|----------|
| Wave ≥ +3 + 月線上方 + 未超倉 | 正常加碼 |
| Wave ≥ +3 但跌破月線 | 等收復月線，不立即加碼 |
| Wave +1～+2 | 輕倉 / 觀察，需搭配持倉定位 |
| Wave 0～-1 | 不加碼 |
| Wave ≤ -2 | Tactical 防守，Core 檢查基本面 |
| Wave ≤ -3 + 跌破月線 / 暫停線 | 波段破壞，優先減持 |

## 波段位置標籤

| 標籤 | 條件 | 意義 |
|------|------|------|
| 🌊 波段初期 | 現價 ≤ 分位數買點中值 | 回測支撐，新一波可能啟動 |
| 🚀 波段中期 | 現價 > 買點，MA ≥ 3/4 | 趨勢延伸，持有最舒服的階段 |
| 🎯 波段後期 | 現價 ≥ 分位數賣出區 | 接近歷史超漲位，提高警戒 |
| ⚖️  波段中性 | 均線混亂，方向不明 | 等待方向確認 |

## 使用方式

```bash
# 單一標的
python scripts/wave_position.py --code 2330

# 多標的比較
python scripts/wave_position.py --code 2330 --code 2317 --code 3455

# 含新建倉預算試算（預算套用至最後一個 --code）
python scripts/wave_position.py --code 3455 --budget 80000

# 三標的同時分析，含由田預算
python scripts/wave_position.py --code 2330 --code 2317 --code 3455 --budget 80000
```
