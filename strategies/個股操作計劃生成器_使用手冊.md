# 個股操作計劃生成器 — 使用手冊

> 框架文件：`strategies/個股操作計劃生成器.md`
> 建立：2026-03-28

---

## 一句話說明

**告訴 AI 你想評估什麼股票，AI 自動跑六步驟，產出完整攻守計劃。**

---

## 三種使用模式

### 模式 A：單檔評估（最常用）

適用：新建倉評估、watchlist 升級決策、持倉重新檢視

**你說：**
```
「幫我跑 2330 的操作計劃」
「聯發科現在適合進場嗎？」
「幫我評估 3037 欣興，要不要從 watchlist 升級」
「力成 Wave 降到 0 了，重新跑計劃」
```

**AI 會做：**
1. 從 `stocks.csv` 取 ticker，跑 `wave_score_scan.py` + `market_state.py`
2. 填寫數據卡（現價、月線、Wave Score、OU 均衡價、GBM μ/σ、殖利率）
3. 三題定性 → 決定 B&H / 波段 / 不碰
4. 計算進場信心分（-5 ~ +5）
5. 設計攻守計劃（加碼批次 + L1/L2/L3 防禦線）
6. GBM 三情境預估獲利 + 風報比

**輸出範例：**

```markdown
## 2330 台積電 操作計劃（2026-03-28）

### 定性結果：B&H 核心倉
- 護城河判斷：更深（全球晶圓代工壟斷）
- 估值判斷：合理（月線折價 -2.4%，GBM +2）
- 腰斬意願：加碼（無條件）
- 適用策略：長線趨勢股交易策略.md

### 進場時機
- 進場信心分：+2.5
- 建議方式：兩批建倉（先 50%，站回月線後補）
- 進場價位：1,820 元
- 進場股數：第一批 5 股（9,100 元）

### 攻擊計劃
| 批次 | 觸發條件 | 股數 | 價位區間 |
|------|---------|------|---------|
| 首批 | Cash ≥ 12% 且 Wave ≥ +2 | 5 股 | 1,800～1,830 |
| 加碼 | Wave ≥ +3 且站回月線 1,864 | 5 股 | 1,860～1,890 |

### 防禦計劃
| 層級 | 觸發條件 | 動作 |
|------|---------|------|
| L1 預警 | Wave ≤ 0 或跌破 1,750 | 停止加碼 |
| L2 減持 | 不適用（B&H 不減持） | — |
| L3 停損 | μ 轉負（極端情境） | 啟動底倉檢討 |

### 預估獲利
| 情境 | 60日 | 1年 |
|------|------|-----|
| 樂觀 (+1σ) | 2,467 (+35.6%) | 4,818 (+164.7%) |
| 基本（期望）| 2,163 (+18.9%) | 3,760 (+106.6%) |
| 悲觀 (-1σ) | 1,859 (+2.2%) | 2,698 (+48.3%) |
| **風報比 (60日)** | **1.89 : 1** | |
```

### 模式 B：全持倉掃描（週末覆盤用）

適用：每週覆盤、大盤劇烈變化後重新評估

**你說：**
```
「跑一次全持倉操作計劃」
「全持倉掃描，更新攻守計劃」
「大盤跌破季線了，重新評估所有持倉」
```

**AI 會做：**
1. 跑全持倉 Wave Score + 大盤狀態
2. 所有持倉依 Wave Score 分為 P0 防禦 / P1 防禦 / P2 攻擊 / P3 觀察
3. 每檔產出數據卡 + 攻守計劃 + GBM 預估
4. 計算資金回收路徑 + Cash 解凍時程
5. 產出行動清單（本週 / Cash 解凍後 / 中期）
6. 存檔為 `strategies/操作計劃_{YYYYMMDD}.md`

**輸出範例：**

```markdown
# 全持倉操作計劃（2026-03-28）

## 總覽
| 優先序 | 分類 | 標的 | 核心行動 |
|--------|------|------|---------|
| P0 防禦 | Wave ≤ -2 | 1215, 3231, 5483 | 保護獲利 / 滾動紀律 |
| P1 防禦 | Wave -1~0 | 6488, 2382, 6115 | 逢反彈出 / 出清 |
| P2 攻擊 | Wave +2 | 2330, 2454 | Cash 解凍後加碼 |
| P3 觀察 | Wave +1 | 6239, 2317, 3455, 2546 | 持有等訊號 |

## 行動清單
| # | 行動 | 標的 | 條件 | 預估影響 |
|---|------|------|------|---------|
| 1 | 出清 | 6115 鎰勝 | μ 負值 | +47,000 Cash |
| 2 | 逢反彈出 | 2382 廣達 | 295+ | +4,400 Cash |
| ... | | | | |
```

### 模式 C：多檔比較（資金有限時）

適用：Cash 剛解凍、多個機會同時出現、只能選一檔

**你說：**
```
「台積電和聯發科，Cash 只夠買一檔，選哪個？」
「2317 鴻海、6239 力成、2454 聯發科，排個優先序」
「Cash 到 12% 了，哪一檔先買？」
```

**AI 會做：**
1. 對所有候選標的跑數據卡 + 定性
2. 計算優先度 = `信心分 × 風報比 × 資金效率修正`
3. 排名輸出，附帶每檔的關鍵差異

**資金效率修正規則：**
- 波段倉（預期 60 天周轉）：× 1.5
- B&H（預期 1 年以上）：× 1.0
- 高配息（殖利率 > 5%）：× 1.2

**輸出範例：**

```markdown
## 加碼優先序比較

| 排名 | 標的 | 信心分 | 風報比 | 資金效率 | 優先度 | 建議金額 |
|------|------|--------|--------|---------|--------|---------|
| 1 | 2330 台積電 | +2.5 | 1.89 | ×1.0 | **4.73** | 9,100 |
| 2 | 2454 聯發科 | +3.5 | 0.43 | ×1.5 | **2.26** | 3,170 |
| 3 | 2317 鴻海 | +2.5 | 0.79 | ×1.0 | **1.98** | 待定 |

結論：台積電優先度最高（護城河 + 風報比雙優），建議先加碼 5 股。
```

|------|------|------|
| 想買新股票 | 「幫我評估 XXXX」 | A 單檔 |
| watchlist 想升級 | 「3037 欣興值不值得建倉？」 | A 單檔 |
| Wave Score 突變 | 「力成 Wave 降到 0 了」 | A 單檔 |
| 收到新聞想判斷 | 「這新聞對持倉影響？重跑相關標的」 | A 單檔 |

| Cash 解凍要選股 | 「Cash 到 12%，排優先序」 | C 比較 |
| 多個機會選一個 | 「這三檔選哪個？」 | C 比較 |

## 注意事項

1. **Cash < 12% 時**：生成器仍會產出計劃，但攻擊計劃會標注「Cash 解凍後才執行」
2. **已定性為減損/出清的持倉**：即使 Wave 訊號轉好，前置檢查仍會擋住加碼建議
3. **零股滾動操作倉（5483、6488）**：不適用一般攻守計劃，會自動切換到滾動 SOP
4. **計劃存檔**：全掃描模式會自動存為 `strategies/操作計劃_{YYYYMMDD}.md`，單檔評估直接在對話中輸出
5. **數據時效**：Wave Score 和 GBM 參數基於最新收盤價，盤中運行可能使用前一日數據
