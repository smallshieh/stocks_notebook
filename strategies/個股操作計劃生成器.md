# 個股操作計劃生成器

> 建立：2026-03-27 | 層級：Layer 3（進出場策略）
> 用途：輸入任一股票代碼 → 讀取現有訊號 → 自動定性 → 產出完整攻守計劃與預估獲利
> 適用：新建倉評估、現有持倉重新檢視、候補股升級決策

---

## 使用流程總覽

```
輸入：股票代碼
        │
Step 1  蒐集數據（自動）
        │
Step 2  三題定性 → B&H / 波段 / 不碰
        │
Step 3  進場時機評估（現在進？等？）
        │
Step 4  攻守計劃（加碼路徑 + 防禦停損）
        │
Step 5  預估獲利與風報比
        │
Step 6  資金優先序（與其他機會比較）
        │
輸出：完整操作計劃
```

## Step 1：數據蒐集（執行清單）

對目標股票執行以下查詢，填入數據卡：

```
□ 從 stocks.csv 取得 ticker
□ 執行 wave_score_scan.py 取得 Wave Score（MA / GBM / 分位 / 物理 / 總分）
□ 執行 market_state.py 取得 OU 參數（均衡價 θ、半衰期、現價偏離%）
□ 從 trades/ 或 watchlist/ 讀取現有持倉資訊（股數、成本、損益%）
□ 查詢基本面：近 4 季 EPS、配息、殖利率、P/E
□ 查詢 GBM 參數：μ（年化漂移率）、σ（年化波動率）
□ 確認月線位置（現價 vs 20MA）
```

## Step 2：三題定性（決定用哪套策略）

### 定性問答

| # | 問題 | 判斷依據 | 答案 |
|---|------|---------|------|
| 1 | 護城河 3 年後更深還是更淺？ | 產業地位、技術壁壘、市佔趨勢 | ⬜ 更深 / ⬜ 更淺 / ⬜ 不確定 |
| 2 | 現在的估值，3 年後回頭看便宜嗎？ | P/E 歷史分位、殖利率位置、GBM μ 方向 | ⬜ 是 / ⬜ 否 / ⬜ 不確定 |
| 3 | 股價腰斬，我願意加碼嗎？ | 對基本面的信心、資金承受度 | ⬜ 是 / ⬜ 否 |

### 定性結果路由

```
三題全「是」
  └→ 🔒 B&H 核心倉 → 套用「長線趨勢股交易策略.md」
       附加條件：GBM μ > +10%

第 3 題「否」或任一題「不確定」
  └→ 🎯 波段倉 → 套用「波段操作策略.md」
       附加條件：必須設停損 + 停利 + 停滯出場

三題中有 2 題以上「否」或「不確定」
  └→ ❌ 不碰 → 放入 watchlist 觀察，不投入資金
       附加條件：設定重新評估日期（30 天後）
```

### 量化輔助判斷

| 指標 | 偏向 B&H | 偏向波段 | 偏向不碰 |
|------|---------|---------|---------|
| GBM μ | > +20% | +5% ~ +20% | < +5% 或負值 |
| σ (波動) | < 40% | 40% ~ 60% | > 60%（除非有操作經驗）|
| 殖利率 | > 4% 且穩定成長 | 2% ~ 4% | < 2% 且無成長 |
| P/E 分位 | < 25%（歷史低位）| 25% ~ 75% | > 75%（歷史高位）|
| OU 偏離 | < -10%（低於均衡）| -10% ~ +5% | > +10%（高於均衡，偏貴）|

## Step 3：進場時機評估（現在進？還是等？）

### 3A. 訊號矩陣（所有訊號匯總）

| 訊號來源 | 看多（+1） | 中性（0） | 看空（-1） |
|---------|-----------|----------|-----------|
| Wave Score | ≥ +3 | +1 ~ +2 | ≤ 0 |
| OU 偏離 | < -10%（便宜） | -10% ~ +5% | > +5%（偏貴）|
| 月線位置 | 站穩且上升 | 剛碰觸 | 跌破且下降 |
| 分位數 | 買回區（≤ 20%）| 中間區 | 偏貴區（≥ 80%）|
| 外資動向 | 連 3 日買超 | 中性 | 連 3 日賣超 |

**進場信心分 = 各項加總（-5 ~ +5）**

| 信心分 | 決策 |
|--------|------|
| +4 ~ +5 | 立即進場，可用較大部位 |
| +2 ~ +3 | 可以進場，建議分批（先 50%） |
| 0 ~ +1 | 觀望，設價格鬧鐘等觸發 |
| -1 ~ -2 | 不進場，加入 watchlist |
| ≤ -3 | 反向操作：若已持有，啟動減持 |

### 3B. 時機選擇（信心分 ≥ +2 時適用）

| 進場方式 | 適用情境 | 說明 |
|---------|---------|------|
| 一次建倉 | 信心分 +4 以上，標的 σ < 35% | 低波動 + 強訊號，不需分批 |
| 兩批建倉 | 信心分 +2 ~ +3 | 先 50%，等確認訊號（站回月線 / Wave 升）再補 |
| 三批建倉 | σ > 50% 高波動股 | 第一批 30%、第二批 40%、第三批 30%，間隔 ≥ 10 交易日 |

## Step 4：攻守計劃（核心輸出）

### 4A. 攻擊計劃（加碼路徑）

依定性結果分開設計：

#### B&H 核心倉攻擊計劃

```
加碼條件：
  ① Wave Score ≥ +3 且月線站穩
  ② 殖利率 ≥ 歷史均值（便宜區確認）
  ③ 距上次加碼 ≥ 20 交易日

加碼節奏：
  每批 = 投組總值 × 1% ÷ 每股風險（capital_config 公式）
  上限 = 單一標的不超過投組 15%

加碼停止條件：
  - P/E 超過歷史 75% 分位
  - 單一標的已達投組 15%
  - Cash 低於 12%（凍結門檻）
```

#### 波段倉攻擊計劃

```
加碼條件：
  ① 第一批持倉已獲利 ≥ 3%（確認方向正確）
  ② Wave Score 維持 ≥ +2（動能未衰退）
  ③ 風報比仍 ≥ 2:1（目標未到、停損未近）

加碼節奏：
  最多兩批：首批 60%、確認後加碼 40%
  距首批 ≥ 5 交易日

加碼停止條件：
  - Wave Score 回落至 ≤ +1
  - 已接近目標價 80%（賣壓區）
  - 持有時間已超過預設的一半（動能消耗）
```

### 4B. 防禦計劃（停損 / 減持）

#### 三層防禦線

| 層級 | 名稱 | 觸發條件 | 動作 | 剩餘部位 |
|------|------|---------|------|---------|
| L1 | 預警線 | Wave Score 降至 ≤ +1 或 OU 回到均衡價 | 停止加碼、提高警覺 | 100% |
| L2 | 減持線 | Wave Score ≤ 0 或月線連 2 日失守 | 賣出波段倉 50% | 減至 ~75% |
| L3 | 停損線 | 持倉虧損 ≥ 10% 或 Wave ≤ -3 | 波段倉全出；B&H 底倉啟動檢討 | 僅剩底倉（B&H）或 0%（波段） |

#### 配息股額外防禦（賣出前必算）

```
價差獲利   = (賣出價 - 成本) × 賣出股數
年配息損失 = 配息/股 × 賣出股數
等於幾年配息 = 價差獲利 ÷ 年配息損失

< 3 年 → 重新考慮（配息損失太大）
3～7 年 → 合理
> 7 年 → 支持出場
```

#### 黑天鵝應變（大盤單日跌幅 > 5% 或週跌 > 10%）

```
啟動條件：加權指數單日跌幅 > 5% 或連續一週跌幅 > 10%

立即動作：
  1. 凍結所有買入計劃（含定期定額），持續至少 5 個交易日
  2. 波段倉：無條件出場（不等反彈）
  3. B&H 底倉：不動，除非個股基本面同步惡化
  4. Cash 目標提升至 20%

恢復條件（全部成立才解凍）：
  - 大盤站回 5 日均線
  - 融資餘額較恐慌日減少 ≥ 10%
  - 至少 3 個交易日無新低
```

## Step 5：預估獲利與風報比

### 5A. GBM 機率情境估算

用 GBM 參數（μ, σ）計算三情境：

```
樂觀情境（+1σ）：
  目標價 = 現價 × exp[(μ - σ²/2) × T + σ × √T]
  預估報酬 = (目標價 - 進場價) / 進場價

基本情境（期望值）：
  目標價 = 現價 × exp[μ × T]
  預估報酬 = (目標價 - 進場價) / 進場價

悲觀情境（-1σ）：
  目標價 = 現價 × exp[(μ - σ²/2) × T - σ × √T]
  預估報酬 = (目標價 - 進場價) / 進場價

T = 持有期（以年為單位，波段倉用 0.25 年 = 60 交易日；B&H 用 1 年）
```

### 5B. 風報比計算

```
攻方（上檔空間）= 基本情境目標價 - 進場價
守方（下檔風險）= 進場價 - 停損價

風報比 = 攻方 ÷ 守方

門檻：
  波段倉 ≥ 2.0 : 1
  B&H 倉 ≥ 1.5 : 1（因持有期長，配息可補償）
```

### 5C. 預估獲利摘要表

```markdown

|------|--------|--------|---------|--------|
```

## Step 6：資金優先序（多個機會時的排序）

當 Cash 有限，多個標的同時觸發進場條件時，依以下公式排序：

### 優先度評分

```
優先度 = 進場信心分 × 風報比 × 資金效率修正

資金效率修正：
  - 波段倉（預期 60 天）：× 1.5（資金周轉快）
  - B&H（預期 1 年）：× 1.0
  - 高配息（殖利率 > 5%）：× 1.2（有現金流回饋）

排序規則：
  1. 優先度最高者先進場
  2. 同分時，風報比較高者優先
  3. 再同分時，Wave Score 較高者優先
```

### 資金門檻檢查（進場前必查）

```
□ Cash 是否 ≥ 12%？
    否 → 凍結，不可新建倉（Layer 1 規則）
    是 → 繼續

□ 本次進場後 Cash 是否仍 ≥ 10%？
    否 → 縮小部位至 Cash 維持 10%
    是 → 按計劃執行

□ 單一標的是否已達投組 15%？
    是 → 不再加碼
    否 → 繼續
```
