# 策略：減損與資金回收運用

> 版本：v1.0（2026-02-26）
> 適用範圍：Tactical_Bucket 為主；Core_Bucket 另有條件

---

## 核心邏輯

- **設計理念**：虧損出場不是終點，而是資金再配置的起點。透過明確的資金分流與再進場條件，避免「停損後茫然不知資金去向」或「無紀律等回彈」兩種極端。
- **適用環境**：持倉出現虧損、需決定全出或部分減倉時；波段滾動標的的操作倉管理。

---

## 一、確認桶別，決定減損態度

| 桶別 | 對減損的態度 | 說明 |
|------|------------|------|
| **Core** | 不以虧損幅度停損 | 以「月線持續失守 + 基本面惡化」為出場條件，分 3～6 個月緩步出清 |
| **Tactical** | 嚴格執行 -10% 首波停損 | 分批出清，明確設定資金去向與再進場期限 |

---

## 二、Tactical 減損觸發條件

**任一成立即啟動減損計畫：**

1. 虧損達 **-10%**（硬停損線，不猶豫）
2. 跌破月線（20MA）且**連續 3 日未收復**
3. 基本面惡化：EPS 預估下修 > 20%、法人集體降評、主力出貨跡象明顯

**排除條件（有一項則暫緩執行）：**
- 虧損係大盤系統性下跌所致（非個股問題）
- 公司公告重大利多（訂單、轉機事件）
- 距除息日 < 30 天且殖利率 > 5%（等領息再評估）

---

## 三、減損出場方式（分批，避免一次砍光）

```
第一批（50%）：觸發條件成立當日或隔日開盤執行
第二批（30%）：等反彈至「成本 -5%」附近的逃命波
第三批（20%）：確認趨勢不逆轉則全數出清；若反彈條件成立則轉為「再進場資金」
```

---

## 四、回收資金分流

### 情境 A：硬停損（不打算回補）

```
回收金額 100% → Cash_Bucket（補充銀彈庫）
```

### 情境 B：暫時減倉（打算等回彈再買）

```
回收金額
  ├── 預留再進場資金（30～50%）→ 留在 Tactical 待命區
  │     └── 設定期限：30～60 天內若未觸發進場條件，自動轉 Cash
  └── 釋出補充（50～70%）→ Cash_Bucket
```

### 情境 C：換股（出清此檔換進另一檔）

```
回收金額在 Tactical 內直接流動，不需過 Cash
但新標的須重新走完進場條件評估流程
```

| 減倉幅度 | 預留再進場 | 釋出補 Cash |
|---------|----------|------------|
| 減 50% | 回收金額的 60% | 回收金額的 40% |
| 減 100%（全出）| 回收金額的 50% | 回收金額的 50% |

---

## 五、再進場條件設定（波段滾動）

> **原則：條件不成立就不回補，超時則放棄，資金轉 Cash。**

### 技術面（優先，任一觸發）
- 收盤重新站上月線（20MA）且成交量放大
- 跌至關鍵支撐區（前波低點、年線、布林下軌）止穩 2 日以上

### 基本面確認（至少一項）
- 法人維持評等或上調目標價
- 當季 EPS 未持續下修

### 排除條件（有一項則放棄回補）
- 月線持續下彎（趨勢反轉，非整理）
- 基本面惡化：獲利下修 > 20%
- 同業股價集體走弱（產業問題而非個股）

### 等待期限
- **標準期限：60 天**
- 超時未觸發 → 預留資金轉移至 Cash_Bucket，等下一個機會

---

## 六、再進場規格（重新計算，不沿用舊參數）

```
再進場股數 = 預留資金 ÷ 新進場成本
新停損線   = 新成本 × 0.9（重設，不繼承舊停損）
新目標價   = 依回彈後的技術位重新設定
```

> ⚠️ 不要用「回到原來成本就好」當目標，要依新的技術結構設定合理的停利區。

---

## 七、決策流程圖

```
持倉出現虧損
      │
      ▼
觸發減損條件？
  │         │
 否          是
  │          │
觀察        確認桶別
            │         │
          Core      Tactical
            │         │
        月線+基本面   分批出清（第一批 50%）
        雙確認才出場
                      │
               有回彈計畫？
               │         │
              是          否
               │          │
          預留 Tactical   100% → Cash
          設期限 60天
               │
         60天內觸發條件？
          │           │
         是            否
          │            │
       分批回補      預留資金 → Cash
      （新參數）
```
