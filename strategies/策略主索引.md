# 策略主索引

> 建立：2026-03-16 | 每月第一個交易日審查，過時策略移入 `archived/`
> **用途**：收到任何訊號或面臨決策時，先查此頁確認用哪個策略、優先順序為何

---

## 決策優先層級

> 衝突時，高層永遠壓過低層。

```
Layer 1  風險邊界      絕對不可違反，無例外
Layer 2  持倉定位      決定你在哪個棋盤上下棋
Layer 3  進出場策略    依持倉定位選對應工具
Layer 4  市場背景修正  調整 Layer 3 訊號的力道
Layer 5  分析工具      提供數據輸入，本身不做決策
```

---

## Layer 1 — 風險邊界（絕對優先）

> 無論其他訊號如何，這些規則永不例外。

| 規則 | 位置 |
|------|------|
| Cash < 10%：禁止建新倉 | `journals/戰術指南.md § 紀律邊界` |
| 停損回收資金不得立即再投入 | 同上 |
| Core 超配 > 50%：不加碼 Core | `strategies/Core部位再平衡建議書.md` |
| P0 出清期間：無條件優先執行 | `journals/戰術指南.md § P0` |

---

## Layer 2 — 持倉定位（決定棋盤）

> 先確認這個倉的身份，再決定用哪個 Layer 3 策略。

| 定位 | 判斷依據 | 對應策略 |
|------|---------|---------|
| 🔒 Core 底倉 | μ > 0、有護城河、長期持有 | `長線趨勢股交易策略.md` |
| ⚙️ 操作倉（零股滾動）| 6488 / 5483 指定操作倉 | `零股滾動法_降成本.md` |
| 🎯 Tactical 倉 | 波段機會、有明確出場計畫 | `波段操作策略.md`（進出場框架）＋ `四維波段評分加減碼策略.md`（Wave Score） |
| 🆕 新倉 / 重新評估 | 輸入股票 → 定性 → 攻守計劃 | `個股操作計劃生成器.md`（整合所有訊號的決策流程） |
| 🚪 減損出清倉 | 深虧 + 策略已定性出清 | `減損與資金回收運用.md` |
| 📊 GBM 分類審查 | 每季重新評估 μ 決定去留 | `GBM量化分類_減持建議_20260311.md` |

**如何判斷定位**：查對應 `trades/*.md` 的「持倉定位」或「減損出清計畫」標題區塊。

## Layer 4 — 市場背景修正（調整訊號力道）

> 這層不做買賣決策，但會調整 Layer 3 的執行力道。

| 策略 | 適用情境 | 修正方式 |
|------|---------|---------|
| `永年_超越時間買賣概念.md` | 市場恐慌 / 大跌時 | 融資未清洗 → 買回區訊號打七折；鳥窩不穩 → 保現金優先 |
| `AI產業趨勢研究_2026Q1.md` | 評估 AI 相關持倉 | 產業背景，補充個股基本面判斷 |
| `個股操作計劃生成器.md § 黑天鵝應變` | 大盤崩跌 >5% / 週跌 >10% | 凍結買入、波段倉無條件出場、Cash 目標提升至 20% |

**當前修正係數（2026-03-16）**：
- 外資 10 天賣超 5,500 億 → 買回區訊號 × 0.7
- 油價 Brent > 100 → 保守係數啟動
- 下次重設時機：融資餘額明顯下降後

3. 若與現有策略重疊，在舊策略頂部加 `> ⚠️ 部分內容已由 {新檔} 取代`
4. 每季審查：確認是否仍適用，不適用移入 `archived/`
