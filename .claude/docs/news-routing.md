# 新聞與資訊消化規範

當用戶提供財經新聞、網路資訊、或由 agent 主動抓取資訊時，依以下規則處理。

## 決策路由（四步驟）

```
收到新聞 / 資訊
    │
    ▼
Step 0：是「已發生事實」還是「第三方前瞻觀點」？
    ├─ 前瞻觀點 → 走前瞻觀點分流：
    │   ├─ 量化型（有條件+時點）→ strategies/thesis_tracking.md 開 entry
    │   ├─ 非量化型 → 試 A 轉化（3-6 月內會看到什麼現象？）
    │   │              ├─ 轉得出 → thesis_tracking 開 entry
    │   │              └─ 轉不出 → 試 C（會改變下個決策嗎？）
    │   │                            ├─ 是 → 修戰術指南觸發條件
    │   │                            └─ 否 → 丟掉
    │   └─ 涉及持倉者 → 同步寫 Step 1 催化劑表（「來源」欄標人名+平台+日期）
    └─ 已發生事實 → Step 1
                │
                ▼
         Step 1：是否與特定持倉/候補股直接相關？
                ├─ 是 → trades/{code}.md 或 watchlist/{code}.md
                │        § 重要事件與催化劑（五欄表格，含「來源」欄）
                └─ 否 → Step 2
                              │
                              ▼
                       Step 2：是否改變操作方向？
                       （觸發條件、停損線、倉位定性）
                              ├─ 是 → 戰術指南.md（直接修改對應觸發條件）
                              └─ 否 → Step 3
                                            │
                                            ▼
                                     Step 3：總經 / 產業訊號
                                            ├─ 盤中 → 戰術指南.md § 當前市況警示（暫存）
                                            └─ 盤後（daily-review 完成後）
                                                      → 盤後日誌.md § 總經與產業訊號
```

## 前瞻觀點收錄門檻（嚴格）

寫入 `strategies/thesis_tracking.md` 的 entry 必須**同時**滿足：

1. **可量化條件**：具體數字、方向、籃子比較（例：「SPX 創歷史新高」、「成長型 AI 籃子總報酬 > 價值型籃子」）
2. **明確驗證時點**：YYYY-MM-DD（法說日、數據發布日、時間窗結束日）

抽象判斷（如「長期看好」、「想像空間巨大」、「會有熊市陷阱」）一律不收，避免淪為剪報。
不收錄但涉及持倉的前瞻觀點 → 只寫進 trades 催化劑表，不必開 entry。

### 非量化前瞻觀點處理（A + C 兩段過濾）

對於有方向但無幅度、無時點的觀點（例：「管理層信心強」、「政策風向偏 AI」、「題材熱度升溫」），**不直接收錄**，依以下兩段過濾：

**Step A — 嘗試量化轉化**

問一句：「**如果這個觀點對了，未來 3-6 個月內我會看到什麼具體現象？**」

- 「管理層信心強」→ 轉成「下次法說不下調指引」或「Q3 EPS 預估上修 ≥ 5%」
- 「政策風向偏 AI」→ 轉成「Q3 前出爐租稅優惠或補貼方案」
- 「題材熱度升溫」→ 轉成「6 個月內該族群指數跑贏大盤 ≥ 10%」

能轉化出可量化條件 + 驗證時點 → **進 thesis_tracking**（在 entry 中註明「由原始定性觀點 X 量化而來」）。

**Step C — 量化失敗則檢查決策影響**

若 Step A 擠不出可量化條件，問第二句：「**這個觀點會不會改變我下一個操作決策？**」（停損線、加碼條件、倉位定性、進出場觸發）

- 是 → 直接修 `journals/戰術指南.md` 對應觸發條件，不另開記錄
- 否 → **丟掉，不記錄**

**禁止**：不開「定性軟驗證區」之類的副表。沒進 thesis_tracking 也沒影響操作的觀點就應該丟掉，記下來就是剪報。

## 時間限制（重要）

| 時段 | 可寫入 | 禁止 |
|------|-------|------|
| 盤前 / 盤中 | `trades/`、`watchlist/`、`戰術指南.md` | ❌ 建立或寫入當日盤後日誌 |
| 盤後（daily-review 執行前）| 同上 | ❌ 建立或寫入當日盤後日誌 |
| 盤後（daily-review 完成後）| 全部，包含盤後日誌 | — |

> **原因**：盤後日誌是 daily-review 的輸出物。盤中搶先建立會在 daily-review 執行時造成內容覆蓋。

## 個股催化劑表格格式

在 `## 重要事件與催化劑` 區塊下，使用**五欄表格**（含「來源」欄）：

```markdown
| 日期 | 事件 | 來源 | 影響評估 | 行動 |
|------|------|------|---------|------|
| 2026-03-14 | Q1 營收 MoM +12% | 公司公告 | 🟢 優於預期 | 維持持有 |
| 2026-04-15 | Q2 可能修正 5-10% | 尼可拉斯楊 YT 4/15 | 🟡 待驗證 | 觀察 7/17 法說 |
```

- **來源欄填寫規則**：
  - 已發生事實 → 「公司公告」/ 「證交所」/ 「MoneyDJ」/ 「鉅亨網」等
  - 前瞻觀點 → 「人名 + 平台 + 日期」（例：「尼可拉斯楊 YT 4/15」、「花旗報告 4/2」）
  - 不確定來源 → 「—」
- 若該 .md 尚無此區塊，在 `## 倉位規劃` 之前插入
- 影響評估用 🔴🟡🟢 標示嚴重程度
- 前瞻觀點寫入時，若已在 `strategies/thesis_tracking.md` 開 entry，「行動」欄可寫「見 thesis_tracking T-XXX」

## 總經訊號格式

```markdown
## 總經與產業訊號
- **Fed**: 鮑威爾暗示年中降息，美債殖利率回落 → 利多成長股
- **半導體**: ASML 訂單指引上修 → 設備股留意
```

## 禁止事項
- **不存原文**：只記消化後的摘要與影響評估
- **不存來源連結**：關鍵字句足以回查，不需要 URL
- **不做無結論的剪報**：每則資訊必須回答「對持倉有什麼影響？需要什麼行動？」
- **不另開 news/ 資料夾**：所有資訊歸入現有結構
