# 🤖 AI 協作最佳實踐 (股票筆記本 SOP)

> ⚡ **重要前提**：持倉健診、淨值記錄、watchlist 掃描已在每個工作日 14:35 **全自動執行**。
> 請先查看當天的 `持倉健診_YYYY-MM-DD.md` 與 `journals/logs/` 的 log，
> 再決定是否需要 AI 介入，避免浪費 tokens。

---

## 📆 每日執行地圖

### 交易日（週一～週五，收盤後）

在 Claude Code 執行：

```
/daily-review
```

> 自動完成：詢問現金餘額或增減金額 → 執行健診 → 追加淨值快照 → 資金桶稽核 → 預警建議 → 建立盤後日誌

### 非交易日（週六、週日、假日）

**無需任何操作。**

---

### 特殊情況

| 情況 | 動作 |
|------|------|
| **存入新現金**（無健診） | 手動在 `portfolio_history.csv` 加一行，填入 `cash_inflow` 金額 |

| **每隔 10 天** | `/new-tactical` → 建立新一期戰術指南 |
| **每月底** | `python scripts/performance_report.py` → 查 TWR / MWR / Alpha |

## 📅 情境一：每日盤後深度檢視

**觸發時機**：自動掃描有預警標的，或想撰寫盤後日誌時。

> **Prompt 範本：**
> 「請依序讀取：
> 1. `capital/capital_config.md`（資金桶定義與稽核規則）
> 2. `持倉健診_今日.md`（現價 / 月線 / 損益）
> 3. `journals/logs/今日_scan.log`（原始掃描紀錄）
>
> 依 capital_config.md 第 5 節執行資金桶稽核，再針對预警標的給出操作建議，
> 最後建立今天的盤後日誌 `journals/YYYY-MM-DD_盤後日誌.md`。」

**💡 AI 會做什麼：**
1. 讀取 `capital_config.md`，核算三桶佔比是否偏離 10 個百分點以上，若有則首行發出 ⚠️ 資金越權警告。
2. 讀取掃描結果，不重複抓取已有的數據。
3. 針對 ⚠️ 停損 / 跌破月線標的給出操作建議（減碼 / 觀察 / 停損）。
4. 建立盤後日誌，填入大盤數據與待辦事項。

## 🔎 情境二：買進前標的分析

**觸發時機**：有感興趣的新標的，想在進場前做完整評估。

> **Prompt 範本：**
> 「我想分析 [股票代號]，請執行 `stock_analyzer.py --ticker [代號]` 取得現價與月線，
> 再幫我查近期法人籌碼動態。若評估偏向進場，請草擬一份 `/trades` 筆記。」

**💡 AI 會做什麼：**
1. 執行腳本取得現價、月線、殖利率（不需 AI 自己搜尋）。
2. 搜尋法人買賣超、外資評等等質化資訊。
3. 使用 `trades/template.md` 建檔，填入原因、停損參考價。

**觸發時機**：想把交易心得量化成可重複執行的策略。

> **Prompt 範本：**

**💡 AI 會做什麼：**

2. 產出 `backtest_[策略名稱].py` 供歷史驗證。

## 📋 情境五：更新 trades 基本資訊空欄位

**觸發時機**：新建持倉筆記後，`目前價格`、`月線 (20MA) 位置`、`預估殖利率` 等欄位仍是空白時。

> **前提**：先確認今日 `journals/logs/YYYY-MM-DD_scan.log` 或 `持倉健診_YYYY-MM-DD.md` 已產生。

> **Prompt 範本：**
> 「請讀取 `持倉健診_今日.md`，將裡面的現價、20MA、殖利率，
> 填進 `trades/` 下各個基本資訊欄位**仍為空白**的 MD 檔。
> 已有數值的欄位不要覆蓋。」

**💡 AI 會做什麼：**
1. 讀取 `持倉健診_YYYY-MM-DD.md`（數字來源，不再重複呼叫 API）。
2. 逐一比對 `trades/*.md`，只更新「目前價格 / 月線 / 殖利率」三個空欄位。
3. 已有值的欄位、策略分析、操作紀錄等內容**完全不動**。

> ⚠️ **注意**：此動作建議每次只在新建持倉後執行一次，後續日常數據以
> `持倉健診_YYYY-MM-DD.md` 為主要看板，不需每日回寫 trades MD。

## 📊 財報狗查詢工作流程

> **使用模式**：你提問 → 我給出「財報狗查詢指令」→ 你去查 → 貼回結果 → 我解讀 + 建議
> 財報狗補足系統的**基本面空缺**，與 GBM / Wave Score 形成技術 + 基本面雙確認。
>
> **網址**：https://statementdog.com/
> **注意**：導覽路徑依欄位名稱自行尋找；負債比/流動比需貼回原始數字由 AI 計算。

---

### 場景 A：Core 持倉季度健診（每季財報後）

**你說**：「請對 [股票代號] 做財報狗季度審查」

**我給出查詢指令**（格式固定如下）：
```
財報狗查詢指令：
1. 前往 財報狗 → 搜尋 [代號] → 點「獲利能力」
   回傳：近 4 季 ROE（%）、近 4 季毛利率（%）
2. 前往 → 點「現金流量」
   回傳：近 4 季自由現金流（正/負，億元）
3. 前往 → 點「每股數據」→ EPS
   回傳：近 4 季 EPS（元）
```

**你貼回格式**：
```
代號：XXXX
ROE：Q1=__, Q2=__, Q3=__, Q4=__（%）
毛利率：Q1=__, Q2=__, Q3=__, Q4=__（%）
自由現金流：Q1=__, Q2=__, Q3=__, Q4=__（正/負）
EPS：Q1=__, Q2=__, Q3=__, Q4=__（元）
```

**我的解讀標準**：
| 指標 | 健康 | 警示 | 退場訊號 |
|------|------|------|---------|
| ROE | > 15%，穩定 | 10-15%，波動 | < 10% 或連續下滑 |
| 毛利率 | 擴張或持平 | 縮減 1-2% | 連續 2 季收縮 > 3% |
| 自由現金流 | 連續正值 | 單季為負 | 連續 2 季為負 |
| EPS | 成長或持平 | 單季衰退 | 連續 2 季衰退 > 20% |

### 場景 B：Tactical 候選股篩選（尋找攻擊子彈）

**你說**：「請給我財報狗篩選條件，找 Tactical 候選股」

**我給出查詢指令**：
```
財報狗查詢指令：
前往 財報狗 → 股票篩選器，設定以下條件：
- 月營收年增率（YoY）> 15%
- 毛利率（最新季）> 30%
- 近 1 季 EPS > 0
- 市值 > 50 億（排除流動性不足）
回傳：符合條件的股票代號清單（前 10-15 檔）
```

**你貼回**：股票代號清單

**我的動作**：挑選其中 Wave Score 適合、與現有持倉低相關者，給出進一步 GBM 分析建議。

### 場景 C：Phase 0 基本面確認（技術 + 基本面雙確認）

**你說**：「[代號] GBM μ 為負，請確認基本面」

**我給出查詢指令**：
```
財報狗查詢指令：
1. 前往 財報狗 → [代號] → 「獲利能力」
   回傳：近 4 季毛利率趨勢（上升/持平/下滑）
2. 前往 → 「月營收」
   回傳：近 3 個月 YoY%（正/負）
3. 在資產負債表找到以下四個原始數字（AI 自行計算比率）：
   回傳：總資產、總負債、流動資產、流動負債（近 4 季）
```

**解讀邏輯**：
| μ | 基本面 | 結論 |
|---|--------|------|
| < 0 | 惡化 | ✅ 雙重確認 → 加速退場 |
| < 0 | 健康 | ⚠️ 市場情緒壓低 → 重新評估，不急退 |
| > 0 | 惡化 | 🚨 技術掩蓋地雷 → 提前預警，縮減倉位 |
| > 0 | 健康 | ✅ 兩面確認 → 正常持有 |

### 場景 D：月營收追蹤（每月 10-15 日）

**你說**：「請給 [代號] 月營收查詢指令」

**我給出查詢指令**：
```
財報狗查詢指令：
前往 財報狗 → [代號] → 「月營收」
回傳：
- 本月營收（億元）
- YoY%（vs 去年同月）
- MoM%（vs 上個月）
- 累計 YoY%（今年 vs 去年同期）
```

**解讀標準**：YoY > +10% 且持續 → 持倉確認；YoY 連 2 月負成長 → 重新評估。

### 場景 E：ETF 持股重疊度檢查（Core 汰換依據）

**你說**：「請比較 [ETF代號A] 和 [ETF代號B] 的重疊度」

**我給出查詢指令**：
```
財報狗查詢指令：
1. 財報狗 → [ETF代號A] → 「持股明細」→ 回傳前 10 大持股
2. 財報狗 → [ETF代號B] → 「持股明細」→ 回傳前 10 大持股
3. 財報狗 → ETF比較頁（若有）→ 費用率、規模（億元）
```

**我的動作**：計算持股重疊比例，結合費用率與規模，給出汰換建議。
