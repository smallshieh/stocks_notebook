## 前提

執行前確認以下條件已就緒：

**1. 健診檔案**
- 優先使用 `持倉健診_{TODAY}.md`
- 若不存在（休市日、假日），往前尋找最近 3 個交易日內的健診檔：
  `持倉健診_{TODAY-1}.md` → `持倉健診_{TODAY-2}.md` → `持倉健診_{TODAY-3}.md`
- 找到後於指南標題註記來源日期，例如：`來源：持倉健診_2026-02-26.md（02-27 休市）`
- 若 3 日內均無檔案 → 提示用戶執行 `python scripts/portfolio_report.py` 並中止

**2. 資金資料**
- `capital/capital_config.md` 的 Cash_Bucket 已填入最新金額
- 若健診檔的 Cash 欄位顯示「請手動填入」，於指南中以最後已知金額代入，並加註：`⚠️ Cash 未更新，使用 {日期} 資料`

**3. Notion 套件（選用）**
- 執行前預先確認：`python -c "import notion_client"` 是否成功
- 若失敗，於流程最後提示：`pip install notion-client`，**不中止主流程**

## 步驟

### Step 1｜備份舊檔

1. 取得今天日期（格式：YYYY-MM-DD），以下以 `{TODAY}` 代稱。
2. 檢查 `journals/戰術指南.md` 是否存在。
3. 若存在，讀取其第一行標題下方的週期資訊，提取起始日期 `{OLD_DATE}`。
   - 重命名為 `journals/戰術指南_{OLD_DATE}.md`
   - 輸出確認訊息：`已備份舊檔 → 戰術指南_{OLD_DATE}.md`

---

### Step 2｜讀取資料來源

依序讀取以下檔案：

1. **`持倉健診_{TODAY}.md`** — 取得所有持倉的現價、20MA、月線位置、損益%、殖利率
2. **`capital/capital_config.md`** — 取得三桶現況佔比、Cash_Bucket 餘額、桶別歸屬表
3. **`.brain/capital_management_rules.md`** — 載入桶別定義、Tactical 佔比上限（35%）、風險計算公式
4. **`trades/*.md`（全部）** — 必須先用 Markdown 結構化工具縮小讀取範圍，不要直接讀完整檔案。
   - 先列出每個檔案的標題與行號：
     ```powershell
     .\.venv\Scripts\python.exe scripts\md_outline.py trades\{檔名}.md
     ```
   - 依 outline 只讀需要的區塊，優先讀：
     `基本資訊`、`倉位規劃`、`操作策略`、`停損預警區`、`減持條件`、`減持計畫`、`零股滾動操作紀錄`、`零股滾動法`、`買賣執行紀錄`、`交易紀錄`
     ```powershell
     .\.venv\Scripts\python.exe scripts\md_section.py trades\{檔名}.md --section "基本資訊" --section "倉位規劃" --section "停損預警區"
     ```
   - 從上述區塊提取：
     - 停損價 / 停利價 / 賣出觸發點 / 買入觸發點
     - 操作倉 vs 底倉股數
     - 待執行策略（DCA 扣款日、零股滾動條件、波段賣出條件等）
     - 進場批次計畫（若有分批建倉）
   - 只有在 outline 缺失、標題歧義、格式損壞，或必要資訊跨多區塊無法定位時，才讀完整 MD。
   - 工具完整說明見 `scripts/MD_TOOLS_FOR_AGENTS.md`。

### Step 3｜識別觸發點與優先序

對每個有明確觸發點的標的，計算「距觸發點百分比」：
```
距觸發% = (觸發價 - 現價) / 現價 × 100%
```
（買入觸發：現價需跌至目標 → 距觸發% 為負；賣出觸發：現價需漲至目標 → 距觸發% 為正）

依距離排序，生成 Priority Stack：

| 等級 | 條件 | 說明 |
|------|------|------|
| P1 | 距觸發 ≤ 3% | 明日極可能觸發，需立即行動 |
| P2 | 距觸發 3～10% | 本期（5 日內）可能觸發，需積極監控 |
| P3 | 距觸發 > 10% | 觀察中 / 按計畫執行（如 DCA）|

#### 3-1｜信心指數評估

對每個觸發點，依以下標準評估信心指數（0～100%）：

**加分因素**
- 觸發點來自 `capital_management_rules.md` 明文規則（+20）
- 該標的過去執行紀錄中此觸發點曾成功執行（+20）
- 基於廣泛使用的技術指標（月線、RSI）（+15）
- 多個獨立訊號同向（距月線 + 成交量 + 籌碼）（+15）

**扣分因素**
- 觸發價首次設定、無歷史依據（-25）
- 觸發價來自外部分析師目標價（未經驗證）（-20）
- 該標的近 3 個月無操作紀錄（陌生行為）（-15）
- 與目前市場環境（Tactical 超限、Cash 不足）相悖（-10）

**信心區間對應行動**

| 信心指數 | 標注 | 行動 |
|---------|------|------|
| > 80% | ✅ 高信心 | 直接列入執行計畫 |
| 65～80% | 🟡 中信心 | 列入計畫，標注不確定性 |
| **< 65%** | **⚠️ 低信心** | **加旗標：依問題類型指定建議驗證方法** |

**強制加旗標條件**（不論信心指數為何）
- 單筆動用資金 > Cash_Bucket 餘額的 30%
- 該標的近 3 個月無任何操作紀錄
- 觸發點為本期首次設定且無歷史依據

**問題類型 → 建議驗證方法對照表**

| 問題類型 | 判斷依據 | 建議驗證方法 |
|---------|---------|------------|
| 觸發點合理性（標的有均值回歸特性） | 觸發價為月線偏離設定 | 均值回歸分析（OU 過程） |
| 趨勢延續機率 | 觸發後預期繼續漲 / 跌 | 趨勢強度估算（GBM + drift）|
| 停損幅度設定 | 停損 % 依經驗而非波動率 | 波動率估算（GARCH）|
| 極端下跌風險 | 停損可能被瞬間穿越 | 極端風險分析（Power law / CVaR）|
| 市場狀態判斷 | 不確定現在是趨勢或震盪 | 市場狀態偵測（HMM）|
| 多標的同時觸發 | 情境試算假設各項獨立 | 相關性分析（Copula）|

旗標格式：
```
⚠️ 信心指數：{n}%（{原因}，建議驗證：{問題類型}）
```

範例：
```
⚠️ 信心指數：58%（觸發點首次設定，建議驗證：均值回歸分析）
⚠️ 信心指數：55%（停損幅度依經驗設定，建議驗證：波動率估算）
⚠️ 信心指數：60%（情境A假設多標的獨立，建議驗證：相關性分析）
```

### Step 4｜建構行動層級（三段式時間架構）

依 Priority Stack 結果，將行動分配至三個時間層級，資訊密度逐層遞減：

**🔫 本週｜`{TODAY+1}` ～ `{TODAY+5}` — 做什麼（細則）**
- P1/P2 項目完整展開：觸發條件、股數、價格、信心指數
- DCA 扣款執行（若扣款日落在本週）
- 需設定的價格提醒

**📋 下週展望 — 準備什麼**
- P2/P3 標的的監控重點（一句話）
- 預期可能觸發的事件（財報、除息日、技術面訊號）
- 需提前準備的資金調度（例：「若本週士電成交，下週可動用回收資金」）

**🧭 下月展望 — 思考與訂計劃**
- 僅列標的名稱 + 一句話方向（例：「鴻海 — 觀察 270 減持時機」）
- 月度 DCA 排程提醒
- 資金桶再平衡方向

### Step 5｜計算銀彈需求

從 `capital/capital_config.md` 取得 Cash_Bucket 現況，計算以下三欄：

**確定支出（無條件執行）**
- 本期 DCA 扣款標的與金額

**條件性支出（觸發後才動用）**
- 各 P1/P2 買入觸發點所需資金（股數 × 觸發價）
- 標注對應桶別（Tactical）

**預期回收（可抵沖支出）**
- 各 P1/P2 賣出觸發點的預計回收金額

**情境試算**
- 情境 A（最積極：所有 P1 + P2 均觸發）
- 情境 B（保守：僅 P1 + DCA，不動 P2）
- 每個情境列出：流出 / 流入 / 淨需求 / 執行後 Cash 剩餘 / Cash 佔比
- 若任一情境執行後 Cash 佔比 < 10%，加入警示：`⚠️ Cash 不足警告`

### Step 6｜設定紀律邊界

列出本期明確的紀律邊界，格式以 `✗` 開頭：

- 底倉持股（Core_Bucket）本期不動（逐一列出標的）
- 月線未收復標的不加碼（列出具體標的）
- Tactical 佔比上限 35% 檢查（依 Step 2 數據核算，若已超標另行警告）
- 各標的停損線（-10% 或月線連跌 2 日，依 trades MD 記錄）
- 本期不新增未列於本指南的任何新標的
