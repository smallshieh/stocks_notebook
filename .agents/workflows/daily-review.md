## 步驟

1. 取得今天日期（格式：YYYY-MM-DD），以下以 `{TODAY}` 代稱。

2. 讀取 `.brain/capital_management_rules.md`，載入三桶定義與稽核規則（第 1 節）及 AI 執行指令（第 5 節）。

3. 確認健診檔案並自動產生（若尚未產生）：
   - 優先讀取 `持倉健診_{TODAY}.md`
   - 若不存在，判斷原因：
     - **今日為週末或國定假日（休市）** → 輸出：`今日休市，daily-review 無需執行。下次執行日：{下一個交易日}。` 並**中止**
     - **今日為交易日但健診未產生** →
       1. 詢問用戶：「請問今日現金餘額是多少元？（輸入數字即可，例如：234498）」
       2. 取得金額後，以 Bash 執行：
          ```bash
          /c/ProgramData/anaconda3/python.exe scripts/portfolio_report.py --cash={現金餘額}
          ```
       3. 執行完成後，讀取新產生的 `持倉健診_{TODAY}.md`，繼續後續步驟（不中止）
       4. 若用戶不知道現金餘額，以 `--cash=0` 執行並在日誌加上 ⚠️ 現金未填註記

4. 依 `capital_config.md` 第 0 節的桶別歸屬表，計算目前 Core / Tactical 市值佔比。
   - 若 Tactical 佔比 > 35%，在回覆**首行**輸出：`⚠️ 資金越權警告：Tactical 超過 35% 上限`

5. 取得預警標的（依序嘗試）：
   - **優先**：讀取 `journals/logs/{TODAY}_scan.log`，提取 ⚠️ 預警行
   - **若 log 不存在**：直接使用步驟 3 健診檔的「需要注意的標的」表格，標注 `（來源：健診檔，無 scan.log）`
   - 兩者均無預警標的時，輸出：`今日無預警標的`

6. 針對每個預警標的，給出以下格式的操作建議：
   ```
   ### [{代號}] {名稱}
   - 現況：現價 vs 月線、損益%
   - 桶別：Core / Tactical
   - 建議：減碼 / 觀察 / 停損（說明理由）
   ```

7. 建立或更新 `journals/{TODAY}_盤後日誌.md`，格式如下：
   ```markdown
   # {TODAY} 盤後日誌
   ## 大盤概況
   （簡短填入，若無資料請留空讓用戶補充）
   ## 預警標的處置
   （貼入步驟 6 的建議）
   ## 待辦事項
   - [ ]（依建議自動生成）
   ```

8. 輸出摘要：今日預警數量、資金桶狀態、已建日誌路徑。
